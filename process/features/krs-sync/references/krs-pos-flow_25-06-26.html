อ่านแผนภาพ:
• ขาเข้า (เขียวทึบ): ดึงสินค้า + ดึงสต็อกอัตโนมัติทุก 5 นาที (sp_Onhand → delta → POS) + หน้า reconcile — ทำงานจริงแล้ว
• หน้าขาย (เขียวทึบ): แคชเชียร์ขาย → คิดเงิน/ตัดสต็อก + จดคิว SyncJob(SALE) ใน transaction เดียว + ออกใบเสร็จ/ใบกำกับภาษี — ทำงานจริงแล้ว
• ขาออก (เหลืองเส้นประ): คิวบิลขายถูกสร้างไว้ แต่ตัวส่งกลับ KRS ยังปิด (flag OFF → 503, ตัวเขียนเป็น stub) — รอ vendor ตอบ 4 จุด (ค่าคงที่ InventoryFlow, คลัง, หน่วยนับ) แล้วจะเปิด Track B เขียน SalesInvoice + TheJournal + InventoryFlow
• วงปิด: เมื่อขาออกเขียน InventoryFlowDtl → sp_Onhand เปลี่ยน → หน้า reconcile ขาเข้ายืนยันให้อัตโนมัติ
