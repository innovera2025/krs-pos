# เอกสารรวมคำถาม/คำขอถึงทีม KRS — ฉบับส่งต่อได้ทันที (17-07-26)

ถึง: ทีมผู้พัฒนา/ผู้ดูแลระบบ KRS (db_ACC_SNP)
จาก: ทีมพัฒนา KRS POS
เรื่อง: คำถามและคำขอที่ค้างอยู่ทั้งหมด เพื่อให้ยอดขายจาก POS เข้าระบบบัญชีครบถ้วน

## 🔴 ส่วนที่ 1 — เร่งด่วน: ปลดล็อกบิลที่มีส่วนลด/โปรโมชัน (ยอดค้างเพิ่มขึ้นทุกวัน)

ปัจจุบัน POS กักบิลที่มีส่วนลดไว้อย่างปลอดภัย (ยังไม่เขียนเข้า KRS) จนกว่าจะได้คำยืนยัน 2 ข้อนี้:

**คำถาม 1.1 (สำคัญที่สุด):** ใน `SalesInvoiceDtl` เราจะส่งค่าแบบนี้ต่อบรรทัด:
- `UnitPrice` = ราคาปกติ (เต็ม)
- `DiscountAmount` = ส่วนลดรวมของบรรทัด (บาท)
- `DiscountPercent` = 0 เสมอ
- `Amount` = ยอดสุทธิบรรทัด = `UnitPrice × Quantity − DiscountAmount`
และผลรวม `Amount` ทุกบรรทัดจะเท่ากับ `SalesInvoiceHdr.TotalAmount` พอดี
→ **ถูกต้องตามที่ KRS คาดหวังหรือไม่? มีหน้าจอ/รายงานตัวใดของ KRS ที่จะนำ `Amount − DiscountAmount` ไปคำนวณซ้ำอีกชั้นหรือไม่?**

**คำถาม 1.2:** สำหรับขายสดที่มีส่วนลด **header (`SalesInvoiceHdr`) ต้องบันทึก field ส่วนลดเพิ่มหรือไม่** (เช่น DiscountAmount ระดับ header)? รบกวนส่ง**ตัวอย่างบิลขายสดที่มีส่วนลด 1 ใบที่ถูกต้อง** (เหมือน workbook ขายสด.xlsx ที่เคยให้) เพื่อเทียบทุกคอลัมน์

**คำขอ 1.3:** ขอ**เป้าทดสอบ (ฐานข้อมูล/ช่วงเวลาที่อนุญาต)** ให้เรายิงบิลทดสอบที่มีส่วนลดชุดเล็ก (~7 ใบ) เพื่อพิสูจน์ความถูกต้องร่วมกันก่อนเปิดใช้จริง — เมื่อผ่านแล้วบิลค้างทั้งหมดจะไหลเข้าระบบเองอัตโนมัติ

## 🟡 ส่วนที่ 2 — คำถามบัญชีที่ควรตอบพร้อมกัน

2.1 บิลมีส่วนลด: GL ที่เราส่ง = 3 บรรทัด (Dr เงินสด = ยอดสุทธิ / Cr รายได้ = ยอดสุทธิก่อน VAT / Cr ภาษีขาย) คือ "รายได้สุทธิหลังหักส่วนลด" — ถูกต้อง หรือฝั่งบัญชีต้องการรายได้เต็ม + บรรทัดส่วนลดแยก?

2.2 ตอนนี้เราส่ง `Receipt_Type` ตามวิธีชำระแล้ว (1=เงินสด 2=โอน 4=บัตร ตามที่แจ้ง) — สำหรับบิล**โอน/บัตร** บรรทัด GL ฝั่งเดบิตควรเป็น**บัญชีธนาคาร**แทนเงินสดหรือไม่ หรือ KRS จัดประเภทเองจาก Receipt_Type? (ปัจจุบันเราเดบิตบัญชีเงินสดทุกบิล)
   - QR PromptPay และ e-Wallet เราส่งเป็นรหัส 2 (โอน) — ถูกต้องหรือควรใช้รหัสอื่น?

2.3 `DiscountPercent = 0` คู่กับ `DiscountAmount` มีค่า → มีระบบใดคำนวณยอดจาก % ซ้ำหรือไม่

2.4 ตาราง `TaxAndDiscount` ต้องมีแถวสำหรับบิลมีส่วนลดหรือไม่ ใครเป็นคนเขียน

2.5 ยืนยันว่า VAT/`SalePurchaseTax.BillAmount` คิดบนยอดหลังหักส่วนลด (ตามที่เราส่งอยู่)

2.6 บิลยอดสุทธิ 0 บาท (แจกฟรี 100%) — KRS รับได้หรือไม่ (ปัจจุบันเรากันไว้ไม่ให้เกิด)

## 🟢 ส่วนที่ 3 — งานฝั่ง KRS (ไม่บล็อกการขาย แต่ควรดำเนินการ)

3.1 **ล้างรหัสสินค้าซ้ำใน item master (~100 กลุ่ม)** — รายการที่ส่งให้ก่อนหน้า (สินค้าเดียวกันหลาย ItemCode): รบกวนเลือกรหัสจริงรหัสเดียวต่อสินค้า (ยึดตัวที่มีการเคลื่อนไหวสต็อก) แล้วปิด/ลบรหัสที่เลิกใช้ — ฝั่ง POS มีระบบตามลบให้อัตโนมัติแล้ว ลบเมื่อไหร่หายจากหน้าขายใน ~3 วินาที

3.2 เพิ่ม **UNIQUE constraint บน `dbo.SalesInvoiceHdr.TransactionNo`** (คำขอเดิม) — ตาข่ายชั้นสุดท้ายกันบิลซ้ำจากระบบใด ๆ

3.3 (ลดระดับจากด่วนเป็นควรแก้) `sp_Onhand @Warehouse=NULL` ตอบ 0 สำหรับสินค้าที่แบบเจาะคลังตอบมียอด (ตัวอย่าง F01-0217 → WH03 = 52 แต่รวมทุกคลัง = 0; พบ ~667 รายการเมื่อ 15-07) — ปัจจุบัน POS เลิกใช้การเรียกแบบรวมแล้ว จึงไม่กระทบเรา แต่ควรตรวจสอบเพราะระบบอื่นที่เรียกแบบรวมจะได้ค่าผิด
