5เริ่มพัฒนา How to start development

ลำดับการลงมือที่แนะนำ:
  1. Commit Phase 1 shell/config/context ถ้ายังไม่ได้ commit (แยก execution กับ process/plan commits)
  2. Review + อนุมัติ HTML timeline นี้ กับทีมก่อนเริ่มโค้ด
  3. ทำทีละเฟส ตามลำดับ dependency (P2 → P7)
  4. ก่อนเริ่มแต่ละเฟส: re-read plan · re-read ทั้งสองไฟล์ design · เช็ค code drift (git diff/log)
  5. ระหว่างเฟส: implement เฉพาะฟังก์ชันของเฟสนั้น · ห้าม drop ฟังก์ชัน Simple POS · รักษาพฤติกรรมเฟสก่อนหน้า
  6. หลังแต่ละเฟส: npm run type-check · npm run build · manual smoke · เขียน report · commit · อัปเดต context ถ้าความรู้ durable เปลี่ยน

6Phase Execution Loop

ลูปบังคับต่อเฟส (จาก process/development-protocols/phase-programs.md)

Research→Approval→Execute→Validate→Regression→Report→Commit→Next phase
  • Research: อ่าน phase plan + design ใหม่, ตรวจ drift
  • Approval: สรุปสิ่งที่จะแก้ + risks + gate, ขออนุมัติก่อนเขียนโค้ด
  • Execute: implement เฉพาะ scope ของเฟส
  • Validate: รัน gate (type-check + build) + manual smoke
