# ⚠️ Reference only — pguard design system (NOT KRS POS source-of-truth)

The files in this `_ds/` directory are the **pguard** (security-guard dispatch platform)
design system. They are included here as **aesthetic / reference material only** — they are
**NOT** the KRS POS product design source-of-truth.

For KRS POS, the canonical sources are:

- **Visual direction:** `design/KRS POS Taste Redesign.html` (the approved Joi/Taste direction)
- **Function inventory:** `design/Simple POS.dc.html` (complete screen/state/flow source-of-truth)

Do **not** treat pguard-specific semantics — guard dispatch, map pins, the guard live-status
triad, etc. — as KRS POS requirements. You may borrow neutral tokens (spacing, type scale,
forest/mint operational palette) for inspiration only, with owner approval.

See `CLAUDE.md` → **UI / design** for the governing rule.
